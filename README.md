# Banking Transfer Information System

Учебный проект по курсу «Разработка информационных систем».  
Веб-приложение для осуществления переводов между банковскими счетами с поддержкой ролей пользователей.  

---

## Описание проекта

Приложение реализует функциональность перевода средств между счетами клиентов банка через веб-интерфейс.  
Система поддерживает четыре роли: клиент, операционист, аналитик и владелец банка.  
Реализованы авторизация, выполнение запросов по клиентам и счетам, формирование отчётов и перевод средств с учётом валютных курсов.  

---

## Стек технологий

- **Backend**: Python, Flask, MySQL  
- **Frontend**: HTML, CSS, Jinja2 Templates  
- **Инструменты**: PyCharm, MySQL Workbench, UML (Use Case, Class, Sequence diagrams)  

---

## Функционал

- Авторизация пользователей по ролям (client / operator / analyst / owner)  
- Работа с запросами:
  - Поиск клиентов по ФИО, дате рождения, адресу, телефону, дате заключения договора  
  - Получение информации о счетах и операциях  
- Работа с отчётами:
  - Формирование SQL-запросов через `reports.json` и вывод данных в интерфейсе  
- Переводы между счетами:
  - Проверка доступного остатка  
  - Конвертация валют по курсу банка на дату операции  
  - Запись в историю операций и журнал переводов

## Описание вариантов использования
### **Вариант использования «Работа с запросами»**

_Предусловие_

Пользователь должен выбрать один из вариантов запроса: «Поиск по номеру телефона / адресу / дате рождения / фамилии / дате подписания и расторжения договора», и «Общий поиск клиентов». Система выдаёт соответствующие результаты из базы данных.

_Минимальная гарантия_

1)	Работа с запросами гарантирует, что БД останется в исходном положении;
2)	Пользователь всегда может вернуться в главное меню системы, оставив БД в согласованном виде;
3)	Пользователь всегда может вернуться на предыдущую страницу.
   
_Сценарий_

1)	Пользователь начинает сценарий;
2)	Система присылает ссылки на варианты запросов (поиск по отдельным параметрам или общий поиск клиентов);
3)	Пользователь выбирает интересующий его запрос (в случае общего поиска пользователь вводит также и параметры поиска);
4)	Система выполняет запрос и выводит html-страницу с результатом;
5)	Пользователь заканчивает работу системы и возвращается в главное меню.
   
_Исключения в сценарии_

4.1. Система сообщает, что введены некорректные параметры; пользователь остаётся на странице общего поиска и может ввести новые параметры (возврат к п. 3);

4.2. Система присылает сообщение об отсутствии данных по данному запросу; пользователь может вернуться на страницу общего поиска или перейти в главное меню.

## Установка и запуск

### Предварительные требования
- Установленный **Python 3.11+**;  
- Установленный **MySQL Server**;
- Установленный **pip**.

### Настройка базы данных
1) Создайте базу данных MySQL, например `bank_transfer_db`;
2) Импортируйте таблицы вручную по схеме проекта;
3) Настройте файл `data_files/.db_config.json` на основе `data_files/.db_config_example.json`.  
   Пример:
```
// data_files/.db_config_example.json

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=bank_transfer_db
SECRET_KEY=your_secret_key
```

### Запуск приложения
```
// bash

# Установка зависимостей
pip install -r requirements.txt

# Запуск сервера
python main.py
```
Приложение будет доступно по адресу: http://localhost:5000/
